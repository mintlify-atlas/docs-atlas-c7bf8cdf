---
title: 'List Sources'
api: 'GET /v1/sources'
description: 'Retrieve all available data source connectors'
---

## Overview

Retrieve the complete catalog of source types that Airweave can connect to, including their authentication methods, configuration requirements, and supported features.

Use this endpoint to discover which integrations are available for your organization.

## Response

Returns an array of source objects.

<ResponseField name="name" type="string" required>
  Human-readable name of the data source (e.g., "GitHub", "Stripe")
</ResponseField>

<ResponseField name="short_name" type="string" required>
  Technical identifier used to reference this source type
</ResponseField>

<ResponseField name="description" type="string">
  Detailed description of what data this source can extract
</ResponseField>

<ResponseField name="auth_methods" type="array">
  List of supported authentication methods (e.g., `direct`, `oauth_browser`)
</ResponseField>

<ResponseField name="oauth_type" type="string">
  OAuth token type for OAuth sources (e.g., `access_only`, `with_refresh`)
</ResponseField>

<ResponseField name="requires_byoc" type="boolean">
  Whether this OAuth source requires users to bring their own client
</ResponseField>

<ResponseField name="auth_fields" type="object">
  Schema definition for authentication fields (only for direct auth)
  
  <Expandable title="properties">
    <ResponseField name="fields" type="array">
      Array of field definitions including name, type, description, and whether it's secret
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="config_fields" type="object" required>
  Schema definition for configuration fields
  
  <Expandable title="properties">
    <ResponseField name="fields" type="array">
      Array of field definitions including name, type, description, default values
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="supported_auth_providers" type="array">
  List of auth provider short names that support this source (e.g., `composio`, `pipedream`)
</ResponseField>

<ResponseField name="labels" type="array">
  Categorization tags (e.g., `code`, `communication`)
</ResponseField>

<ResponseField name="supports_continuous" type="boolean" required>
  Whether this source supports cursor-based continuous syncing
</ResponseField>

<ResponseField name="federated_search" type="boolean" required>
  Whether this source uses federated search instead of syncing
</ResponseField>

<ResponseField name="supports_access_control" type="boolean" required>
  Whether this source supports document-level access control
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl https://api.airweave.ai/v1/sources \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

url = "https://api.airweave.ai/v1/sources"
headers = {"Authorization": "Bearer YOUR_API_KEY"}

response = requests.get(url, headers=headers)
sources = response.json()

for source in sources:
    print(f"{source['name']} ({source['short_name']})")
```

```typescript TypeScript
const url = 'https://api.airweave.ai/v1/sources';
const headers = {
  'Authorization': 'Bearer YOUR_API_KEY'
};

const response = await fetch(url, { headers });
const sources = await response.json();

sources.forEach(source => {
  console.log(`${source.name} (${source.short_name})`);
});
```

</CodeGroup>
