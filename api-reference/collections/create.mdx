## Overview

Create a new collection to organize and search across data from multiple sources. After creation, add source connections to begin syncing data.

The collection will be assigned a unique `readable_id` based on the name you provide, which is used in URLs and API calls.

## Request Body

<ParamField body="name" type="string" required>
  Human-readable display name for the collection (4-64 characters)
  
  Example: "Finance Data", "Customer Support"
</ParamField>

<ParamField body="readable_id" type="string">
  Custom URL-safe identifier. If not provided, auto-generated from name.
  
  Must contain only lowercase letters, numbers, and hyphens.
  
  Example: "finance-data-reports"
</ParamField>

<ParamField body="sync_config" type="object">
  Default sync configuration for all syncs in this collection
  
  <Expandable title="properties">
    <ParamField body="handlers" type="object">
      <ParamField body="enable_vector_handlers" type="boolean" default={true}>
        Enable vector database handlers
      </ParamField>
      <ParamField body="enable_postgres_handler" type="boolean" default={true}>
        Enable PostgreSQL metadata handlers
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns the created collection object.

<ResponseField name="id" type="string" required>
  Unique UUID identifier for the collection
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable display name
</ResponseField>

<ResponseField name="readable_id" type="string" required>
  URL-safe unique identifier
</ResponseField>

<ResponseField name="status" type="string" required>
  Initial status will be `NEEDS_SOURCE`
</ResponseField>

<ResponseField name="vector_size" type="integer" required>
  Vector dimensions used by this collection
</ResponseField>

<ResponseField name="embedding_model_name" type="string" required>
  Name of the embedding model used
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp when created
</ResponseField>

<ResponseField name="organization_id" type="string" required>
  UUID of your organization
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.airweave.ai/v1/collections \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Finance Data",
    "readable_id": "finance-data-reports"
  }'
```

```python Python
import requests

url = "https://api.airweave.ai/v1/collections"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "name": "Finance Data",
    "readable_id": "finance-data-reports"
}

response = requests.post(url, headers=headers, json=data)
collection = response.json()
```

```typescript TypeScript
const url = 'https://api.airweave.ai/v1/collections';
const headers = {
  'Authorization': 'Bearer YOUR_API_KEY',
  'Content-Type': 'application/json'
};
const data = {
  name: 'Finance Data',
  readable_id: 'finance-data-reports'
};

const response = await fetch(url, {
  method: 'POST',
  headers: headers,
  body: JSON.stringify(data)
});
const collection = await response.json();
```

</CodeGroup>

## Error Responses

<ResponseField name="400 Bad Request">
  Collection with the specified readable_id already exists
</ResponseField>

<ResponseField name="422 Validation Error">
  Invalid request body (e.g., name too short, invalid readable_id format)
</ResponseField>
